## Getting started

```bash
# 1. Python workspace
make sync             # uv sync --all-extras

# 2. Secrets — drop your real keys into .env
cp .env.example .env
#   fill in ANTHROPIC_API_KEY, OPENAI_API_KEY, LANGFUSE_*

# 3. Bring up the dev stack (postgres, redis, litellm, langfuse)
make up
make ps               # confirm everything is healthy

# 4. Apply the initial schema
make migrate          # alembic upgrade head

# 5. Run the full quality gate (mirrors CI)
make check
```

`make check` runs: ruff lint, ruff format check, mypy (strict), and pytest. Everything should be green on a fresh clone.

## Everyday commands

| Command | What it does |
|---|---|
| `make help` | Show all commands |
| `make sync` | Sync the workspace |
| `make up` / `make down` | Start / stop the dev stack |
| `make logs` | Tail every service's logs |
| `make db-shell` | `psql` into the meridian DB |
| `make migrate` | `alembic upgrade head` |
| `make migrate-new MSG="..."` | Create an autogenerated migration |
| `make check` | Full CI-equivalent quality gate |
| `make reset` | Stop everything and wipe volumes |

## Running the quality gate

```bash
make lint      # ruff check
make fmt       # ruff format (writes)
make fmt-check # ruff format --check
make type      # mypy, strict, in package mode
make test      # pytest
make check     # all of the above
```

CI runs the same commands on every PR — see `.github/workflows/ci.yml`.

## Architectural decisions locked in for v1

These come from Section 19 of the execution plan. If you want to change one, read that section first.

- **Multi-provider via LiteLLM gateway** (D1, D2) — Anthropic + OpenAI, with Anthropic as priority 1 for every tier
- **Prompt registry in Postgres** (D3) — versioned, immutable rows; rollback via activation flip
- **Three-tier model routing** (D4) — `meridian-small` / `meridian-mid` / `meridian-frontier`
- **Hand-rolled deterministic state machine** (D5) — no LangGraph/Temporal in v1
- **RAG consumed, not built** (D6) — retrieval contract is in `meridian_contracts.retrieval`
- **Synchronous tool execution** (D7) — async is a v2 decision
- **Self-hosted Langfuse** (D9) — v3 stack (web + worker + clickhouse + minio)
