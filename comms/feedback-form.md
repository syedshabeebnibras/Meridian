# Early-user feedback form

For beta testers and early adopters during the dogfood / 25% rollout
windows. Deliver via Google Form or Slack workflow. Questions below are
the minimum set — add company-specific context as needed.

---

**Meridian feedback — [Cohort name] — [Date range]**

### About you

1. What team are you on?
2. How many Meridian queries did you run this week?  (0 / 1–5 / 6–20 / 21+)

### Quality

3. Of the answers you got this week, roughly what share felt **correct
   and useful**?  (None / A few / About half / Most / Almost all)

4. Did Meridian **hallucinate** anything — i.e. assert something that
   turned out to be wrong or not in the cited source?  If so, please
   share the query:

5. Did Meridian **refuse** when you thought it should have answered?
   If so, what were you asking?

### Speed

6. How did response time feel?
   (Fast / Acceptable / Slow / Unusably slow)

### Tool actions

7. Did you use the Jira or Slack tool actions? If so, did the
   confirmation flow feel right, or did it get in the way?

### The one thing

8. **What's the one thing that would make Meridian more useful to
   you?** (One-liner is fine.)

### Open feedback

9. Anything else? Praise, pain points, bugs, feature requests.

---

**Incentive:** a thank-you for every response. No scoring, no grading —
we want frank feedback.

Responses feed directly into the Monday weekly review (Section 11).
